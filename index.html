ExcellenSight: AI-Powered ChatGPT Feedback Analyzer
View on GitHub

ExcellenSight is a robust, end-to-end natural language processing system designed to extract actionable insights from extensive user feedback on ChatGPT. At its core is a custom neural network model developed from scratch, intentionally avoiding any reliance on pre-trained architectures. This multi-task network consists of a shared encoder with two task-specific heads: one for classifying feedback into five well-defined trend categories (such as pain points or praise), and another for summarizing key messages into coherent, sentence-level insights. This combination enables nuanced understanding of public sentiment toward ChatGPT.

The pipeline begins by ingesting thousands of raw user comments and applying thorough data preprocessing techniques. It then trains the neural model on these cleaned datasets while monitoring classification accuracy and summarization quality. ExcellenSight automatically compiles professional-grade reports that include not only text-based trend summaries but also rich visualizations such as training loss graphs and frequency bar charts. These reports are generated in both Markdown and HTML, allowing for flexible sharing and integration.

To enhance usability, the platform is deployed as a Streamlit application that supports real-time interaction. Users can upload new datasets, inspect categorized reviews, and explore generated insights dynamically through the app interface. The entire system is engineered to be highly scalable, capable of processing tens of thousands of reviews for large-scale sentiment analysis. ExcellenSight stands out for its ability to transform raw textual feedback into data-driven insights that can inform UX design, marketing, and product improvements.

Taskify: AI-Powered Task-to-Team Member Matching System
View on GitHub

Taskify is a sophisticated AI-driven recommendation engine that automates the assignment of organizational tasks to the most suitable team members. It leverages a custom Transformer-based neural network that integrates both semantic information (from task descriptions) and structured features (from team member profiles) to predict compatibility with high precision. The architecture is designed with a hybrid attention mechanism that enables contextual understanding of both the task and the team landscape.

To simulate real-world diversity, the training dataset consists of over 40,000 synthetic examples representing task-member pairs from different departments, roles, and expertise levels. The model is trained as a binary classifier that outputs a compatibility score, helping prioritize task allocation based on predicted effectiveness. The inclusion of cross-attention layers ensures that relational nuances between task requirements and individual qualifications are well captured.

Validation is an integral part of Taskify's workflow, featuring a full suite of evaluation metrics such as accuracy, precision, recall, F1-score, and confusion matrices. These tools help monitor performance across various organizational settings. For usability, Taskify includes a local Streamlit interface that allows real-time input of task descriptions and displays the top recommended team members, complete with their fit scores. This functionality makes the model directly actionable in a project management setting. Taskify's adaptability, depth of inference, and real-time deployment capabilities make it an impactful tool for enhancing collaboration and efficiency in team-based environments.

GDELT Monitoring System: LLM-Powered Global Macro Investment Engine
View on GitHub

This project simulates the decision-making workflow of a portfolio manager at a Global Macro Hedge Fund, using a fully local LLM (Large Language Model) to generate investment strategies based on real-time global news. Built around the GDELT database, the system scrapes live geopolitical event data, processes and cleans it, stores it in a vector database, and uses prompt-engineered LLM reasoning to recommend country-level long or short financial positions.

The architecture includes dedicated Python modules for scraping data from the GDELT website, transforming it for NLP pipelines, and embedding text data into a Chroma vector database. Exploratory analysis and vector visualizations are performed using PCA and t-SNE. The investment decision engine uses a locally hosted LLaMA 3.2B model through Ollama, enabling privacy-preserving inference completely offline. By using prompt engineering and text embeddings, the LLM summarizes recent geopolitical trends and recommends trade actions at the country level.

The entire pipeline can be executed end-to-end locally via Python scripts, and is structured for modular testing and scalability. This project highlights a unique blend of real-world data aggregation, machine learning infrastructure, and financial analysis powered by AI, showcasing the application of generative models in global finance and investment research.

Rock-Paper-Scissors Recognition on ESP32S3
Demo Not Available

This project demonstrates a proof-of-concept vision-based rock-paper-scissors game designed to run on the ESP32S3 Sense microcontroller. As part of a systems integration pitch for a next-generation interactive game device, I developed a CNN-based hand gesture classifier to recognize rock, paper, and scissors signs from a camera feed using only onboard hardware and Python/MicroPython.

The project focused solely on recognizing the hand gestures, not implementing the full game logic. The ESP32S3 served as both the image acquisition and inference engine. All training data was collected using the onboard camera, cleaned and augmented, and later used to train a compact convolutional neural network tuned for resource-constrained environments.

Images were captured at 96x96 resolution and resampled to 32x32 bitmaps to reduce memory usage. The pre-processing pipeline involved basic image transformations and was kept lightweight to match the ESP32's limited capabilities. Techniques like edge detection and thresholding were considered but avoided where possible due to compute constraints.

To improve generalization, I gathered several hundred clean images per class directly from the ESP camera, using a custom socket server/client setup. I then applied augmentation and trained the CNN on a laptop. The final model achieved 56% classification accuracy when deployed back onto the ESP device using MicroPython.

Key takeaways included navigating embedded deployment constraints, applying image processing creatively, and validating the use of affordable microcontrollers in real-world vision tasks. This project served as a technical feasibility test and proof of execution for future gesture-based consumer products.
